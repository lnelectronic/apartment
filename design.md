| col | A | B | C | D | E | F |

### Formula สำหรับ "ค้างเดือนก่อน" (col 13)

เมื่อ Apps Script เขียน row ใหม่สำหรับห้อง RoomX เดือน M/YYYY จะใส่ formula:

```
=IFERROR(
  VLOOKUP(PREV_MONTH&"|"&RoomX, 'บันทึก'!$A:$B&"|"&..., รวม_col, 0)
  - VLOOKUP(PREV_MONTH&"|"&RoomX, ..., จ่ายจริง_col, 0),
  0
)
```

**วิธีง่ายกว่า — ใช้ SUMPRODUCT:**
```
=IFERROR(
  SUMPRODUCT(('บันทึก'!$A:$A=PREV_MONTH)*('บันทึก'!$B:$B=THIS_ROOM)*'บันทึก'!$S:$S)
  -SUMPRODUCT(('บันทึก'!$A:$A=PREV_MONTH)*('บันทึก'!$B:$B=THIS_ROOM)*'บันทึก'!$T:$T),
  0
)
```

- `PREV_MONTH` = เดือน/ปี ก่อนหน้า (คำนวณจาก cell col A ของ row นี้)
- `THIS_ROOM` = เลขห้อง (col B ของ row นี้)
- col S = รวม (col 19), col T = จ่ายจริง (col 20)

## Web App Routing

```
doGet(e):
  page = e.parameter.page
  if page == "admin" → serve admin.html (ต้องใส่ password)
  else               → serve staff.html (default)
```

### หน้าพนักงาน (staff.html) — ?page=staff
1. กรอกเลขห้อง
2. ระบบ lookup ใน "ตั้งค่า" — ไม่พบ → error
3. ถ้าพบ + ยังไม่มีข้อมูลเดือนนี้ → ดึง "ถึง" เดือนก่อนมาเป็น "เริ่ม" (lock)
4. ถ้ากรอกไปแล้ว → ถาม [แก้ไขทับ] หรือ [ยกเลิก]
5. กรอกมิเตอร์ "ถึง" น้ำ/ไฟ + รายการอื่นๆ (ไม่เกิน 2)
6. กดคำนวณ → แสดงยอดรวม → กดบันทึก (เห็นข้อมูลการเงินครบ)
7. แก้ไขทับ: อัปเดตมิเตอร์และยอด ห้ามแตะ "จ่ายจริง" และ "สถานะ"

### หน้าพนักงานจดมิเตอร์ (meter.html) — ?page=meter
1. กรอกเลขห้อง
2. ไม่พบ → error / จดแล้วเดือนนี้ → บล็อก (ให้ติดต่อแอดมิน)
3. แสดง: ชื่อผู้เช่า + สถานะ (ปกติ/ค้างจ่าย) + มิเตอร์เริ่ม (readonly)
4. กรอกมิเตอร์ถึง + รายการอื่นๆ → บันทึกเลย (ไม่มี step คำนวณ ไม่เห็นยอดเงิน)

### หน้าแอดมิน (admin.html)
- Login ด้วย password (เปรียบเทียบกับ Script Properties)
- Dashboard: summary เดือนปัจจุบัน + ตารางค้าง + แจ้งเตือนห้องที่ยังไม่กรอกมิเตอร์
- กรอก "จ่ายจริง" และ "ค่าปรับ" รายห้อง
- ยืนยัน "จ่ายแล้ว" (auto เมื่อ จ่ายจริง >= รวม)
- ดูบิล print-friendly ต่อห้อง (แชร์ LINE)
- จัดการห้อง: แก้ชื่อผู้เช่า, ค่าเช่า, ค่าเฟอร์นิเจอร์ใน "ตั้งค่า"

## เดือน/ปี Format

- เก็บเป็น string `"M/YYYY"` เช่น `"6/2026"`, `"12/2026"`
- ไม่ใช้ zero-pad (`"06/2026"`) เพื่อให้ compare ง่ายและ human-readable
- PREV_MONTH คำนวณโดย: แยก M กับ YYYY ออก, ถ้า M=1 → PREV = "12/(YYYY-1)", อื่นๆ → "(M-1)/YYYY"

---

## Admin Auth Flow

1. `admin.html` โหลด → แสดง password form
2. submit → `google.script.run.checkPassword(pw)` → server เปรียบเทียบกับ `PropertiesService.getScriptProperties().getProperty('ADMIN_PASSWORD')`
3. ถ้าผ่าน → แสดง Dashboard (เก็บ session ใน `sessionStorage` ฝั่ง client)
4. ถ้าไม่ผ่าน → error message

## Deploy Steps (สำหรับ session ถัดไป)

```bash
# 1. สร้าง Spreadsheet และ bound script ผ่าน clasp
cd src
clasp create --title "App-Room" --type sheets

# 2. push โค้ด
clasp push

# 3. ตั้ง Admin Password
# ทำใน Apps Script Editor: File > Project Properties > Script Properties
# key: ADMIN_PASSWORD, value: <รหัสที่ต้องการ>

# 4. Run SheetSetup() ครั้งเดียวเพื่อสร้าง Sheets + ใส่ข้อมูลเริ่มต้น

# 5. Deploy as Web App
# Execute as: Me, Who has access: Anyone
```

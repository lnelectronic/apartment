# Requirement: ระบบคำนวณค่าเช่าห้องพัก (Google Apps Script + Google Sheets)

## เป้าหมาย

สร้างระบบคำนวณค่าเช่า/ค่าน้ำ/ค่าไฟห้องพัก ใช้ Google Sheets เป็นฐานข้อมูล และ Google Apps Script
เป็น Web App ให้พนักงานกรอกข้อมูล แทนระบบเดิมที่เสียค่าบริการรายปี
Deploy ผ่าน Google clasp (CLI)

## โครงสร้าง Google Sheets

### Sheet 1: "ตั้งค่า"

#### ส่วนที่ 1: ราคาสาธารณูปโภค (แยกตามตึก)

| ตึก | ราคาน้ำต่อหน่วย | ราคาไฟต่อหน่วย |
|-----|----------------|----------------|
| A   | ...            | ...            |
| B   | ...            | ...            |

ตึก A และ B คิดราคาน้ำ/ไฟต่างกันได้ ระบบดึงราคาจาก prefix ของเลขห้อง (A→ชุด A, B→ชุด B)

#### ส่วนที่ 2: รายชื่อห้องทั้งหมด (Pre-created ก่อน deploy)

คอลัมน์: **เลขห้อง | ชื่อผู้เช่า | ค่าเช่า | ค่าเฟอร์นิเจอร์**

- เลขห้องใช้ prefix ตึก เช่น `A201`, `A301`, `B201`, `B301`
- ห้องทั้งหมดถูก pre-create ไว้ก่อน — พนักงานไม่สามารถเพิ่มห้องใหม่เองได้
- ค่าเฟอร์นิเจอร์ = fixed รายห้อง (บางห้องมี บางห้องไม่มี ใส่ 0 ถ้าไม่มี)
- ถ้าเปลี่ยนผู้เช่า แอดมินแก้ชื่อในช่องนี้เพียงจุดเดียว

### Sheet 2: "บันทึก" (ฐานข้อมูลหลัก เก็บทุกห้องทุกเดือน)

**เดือน/ปี ของแต่ละ row = เดือนที่ผู้เช่าต้องจ่าย** (ไม่ใช่เดือนที่จดมิเตอร์)
ตัวอย่าง: จดมิเตอร์เดือน 5 → row บันทึกเป็นเดือน 5 รวมค่าเช่าเดือน 5 + ค่าน้ำ/ไฟที่ใช้ในเดือน 4

คอลัมน์:

* เดือน/ปี, เลขห้อง
* ไฟ: เริ่ม, ถึง, ใช้ไป, เป็นเงิน
* น้ำ: เริ่ม, ถึง, ใช้ไป, เป็นเงิน
* ค่าเช่า (ดึงจาก "ตั้งค่า" อัตโนมัติ)
* ค่าเฟอร์นิเจอร์ (ดึงจาก "ตั้งค่า" อัตโนมัติ)
* ค้างเดือนก่อน — **dynamic formula**: = รวมเดือนก่อน − จ่ายจริงเดือนก่อน (คำนวณสดทุกครั้ง ไม่ lock ค่า)
* ค่าปรับ (แอดมินกรอกเองตอนสรุปบิล ไม่ auto คำนวณ — carry over อัตโนมัติผ่าน ค้างเดือนก่อน ถ้ายังไม่จ่าย)
* รายการอื่นๆ + จำนวนเงิน (ช่องเพิ่มเติม กรอกเป็นรายการได้)
* รวม = ไฟ + น้ำ + ค่าเช่า + ค่าเฟอร์นิเจอร์ + ค้างเดือนก่อน + ค่าปรับ + รายการอื่นๆ (คำนวณอัตโนมัติ)
* **จ่ายจริง** (แอดมินกรอกยอดที่รับเงินมาจริง — กรอกได้เฉพาะแอดมิน)
* สถานะการจ่าย (จ่ายแล้ว/ค้าง) — เปลี่ยนเป็น "จ่ายแล้ว" ได้เฉพาะเมื่อ จ่ายจริง ≥ รวม

### Sheet 3: "Dashboard"

แสดงข้อมูลของ **เดือน/ปีปัจจุบัน** (ดึงจาก `new Date()` อัตโนมัติ)

**Summary:**
* จำนวนห้องทั้งหมด / จ่ายแล้ว / ยังไม่จ่าย
* **ยอดที่ควรเก็บได้** = sum(รวม) ทุกห้องเดือนนี้ รวม carryover = เก็บได้จริง + ค้างรวม เสมอ
* **เก็บได้จริง** = sum(จ่ายจริง) ทุกห้องเดือนนี้
* **ค้างรวม** = sum(รวม − จ่ายจริง) ทุกห้องเดือนนี้

**ตารางห้องที่ยังไม่จ่าย:**

| เลขห้อง | ผู้เช่า | ยอดเดือนนี้ | ยอดค้างสะสม | จำนวนเดือนที่ค้าง |
|---------|--------|------------|------------|-----------------|

* **ยอดเดือนนี้** = ค่าใช้จ่ายใหม่เท่านั้น (ไฟ + น้ำ + ค่าเช่า + ค่าเฟอร์นิเจอร์ + รายการอื่นๆ) ไม่รวม ค้างเดือนก่อน
* **ยอดค้างสะสม** = รวม − จ่ายจริง (ยอดจริงที่ต้องเก็บ รวม carryover ทั้งหมด)
* **จำนวนเดือนที่ค้าง** = นับเดือนติดกันย้อนหลังจากเดือนปัจจุบัน จนกว่าจะเจอเดือนที่ (รวม − จ่ายจริง) = 0

**แจ้งเตือน:**
* ห้องที่ยังไม่กรอกมิเตอร์เดือนนี้ (เปรียบเทียบรายชื่อห้องใน "ตั้งค่า" กับ row ที่มีในเดือนปัจจุบัน)

## Web App (Apps Script) — แยก 2 ลิงก์

### ลิงก์ 1: หน้าพนักงาน (มือถือ, ไม่ login, ไม่เห็นข้อมูลห้องอื่น)

Flow:

1. พิมพ์เลขห้อง (format: A201, B301 ฯลฯ)
2. ระบบเช็ค:

   * ห้องมีอยู่ใน "ตั้งค่า" + ยังไม่กรอกเดือนนี้ → ดึง "เลขถึง" มิเตอร์เดือนก่อนมาเป็น "เริ่ม" อัตโนมัติ (ล็อกแก้ไม่ได้), ดึง ค้างเดือนก่อน (dynamic) มาแสดง
   * ห้องมีอยู่ + กรอกเดือนนี้ไปแล้ว → เตือน "มีข้อมูลแล้ว" ให้เลือก \[แก้ไขทับ] หรือ \[ยกเลิก]
   * **ไม่พบเลขห้องในระบบ → แสดง error "ไม่พบห้องนี้ กรุณาติดต่อแอดมิน"** (ไม่มี flow เพิ่มห้องใหม่จากหน้านี้)

3. กรอกเลขมิเตอร์ "ถึง" ของน้ำ/ไฟ (เลขที่จดมาวันนี้)
4. กรอกรายการอื่นๆ ได้ (ชื่อรายการ + จำนวนเงิน)
5. ไม่มีช่องค่าปรับ / จ่ายจริง / สถานะ ในหน้านี้
6. กดคำนวณ → แสดงยอดรวม (ไม่รวมค่าปรับ) → กดบันทึกเข้า Sheet "บันทึก"
7. **เมื่อแก้ไขทับ**: อัปเดตเฉพาะมิเตอร์และยอดคำนวณ — **ค่า "จ่ายจริง" และ "สถานะ" คงไว้ไม่เปลี่ยน**
8. ทำทีละห้อง (1 ห้องต่อครั้ง เดินจดมิเตอร์ทีละห้อง)

### ลิงก์ 2: หน้าแอดมิน (Dashboard + จัดการบิล)

* ดู Dashboard สรุปทั้งหมด (เดือนปัจจุบัน)
* เพิ่ม/แก้ **ค่าปรับ** รายห้อง
* กรอก **จ่ายจริง** เมื่อได้รับเงิน (กรอกได้เฉพาะแอดมิน)
* กดยืนยัน "จ่ายแล้ว" เมื่อ จ่ายจริง ≥ รวม เท่านั้น
* ถ้าจ่ายไม่เต็ม ยังคงสถานะ "ค้าง" และยอดที่ขาดพ่วงไปเดือนถัดไปอัตโนมัติผ่าน dynamic formula
* **พิมพ์/แชร์บิลรายห้อง** — แอดมินกดดูบิล print-friendly ต่อห้อง แล้วพิมพ์หรือ screenshot แชร์ LINE ผู้เช่าได้
* จัดการห้อง: เพิ่มห้องใหม่, แก้ชื่อผู้เช่า, แก้ค่าเช่า/ค่าเฟอร์นิเจอร์ใน "ตั้งค่า"

## การตั้งค่า Deploy

* Web App "Execute as": Me (เจ้าของบัญชี/Sheet) — เพื่อให้ทุกคนที่กรอกผ่านลิงก์ บันทึกเข้า Sheet เดียวกันของเจ้าของห้องพัก ไม่ว่าจะใช้ Gmail คนละบัญชี
* เจ้าของห้องพัก = บัญชีที่สร้างสคริปต์ + เป็นเจ้าของ Sheet
* แอดมิน = ได้รับสิทธิ์ Editor ของ Sheet (หรือใช้ลิงก์แอดมินแยก)
* พนักงาน = ใช้แค่ลิงก์ Web App ไม่ต้องมีสิทธิ์เข้า Sheet

## เครื่องมือที่จะใช้

* Google Apps Script (.gs files) + HTML Service (.html สำหรับหน้าเว็บ)
* Google clasp (CLI) สำหรับ push/deploy โค้ดจากเครื่อง local

## ขั้นตอนการพัฒนา → ส่งมอบให้เจ้าของห้องพัก

ผู้พัฒนา (ฉัน) ไม่ใช่เจ้าของห้องพัก และจะไม่ขอรหัสผ่าน/login เข้าบัญชี Gmail ของเจ้าของห้องพักเด็ดขาด
ลำดับงานที่จะทำ:

1. **ช่วงพัฒนา/ทดสอบ**: สร้าง Google Sheet ใหม่ด้วย Gmail ของผู้พัฒนาเอง, เขียน Apps Script,
ทดสอบด้วยข้อมูลปลอม (ห้องสมมติ), deploy ทดสอบเอง (Execute as: Me = ผู้พัฒนา)
2. **ก่อนส่งมอบจริง**:

   * ลบข้อมูลทดสอบทั้งหมดออกจาก Sheet "บันทึก" ให้เหลือแต่โครงสร้างคอลัมน์เปล่า
   * โอนความเป็นเจ้าของไฟล์ (Transfer Ownership) ให้ Gmail ของเจ้าของห้องพัก ผ่าน Share > เปลี่ยนสิทธิ์เป็น Owner
   * เจ้าของห้องพักต้องกด Accept รับโอนความเป็นเจ้าของ
   * Apps Script ที่ผูกกับ Sheet (container-bound script) จะโอนเป็นของเจ้าของห้องพักไปด้วยอัตโนมัติ
3. **เจ้าของห้องพักกด Deploy เอง** (1 ครั้ง หลังรับโอนไฟล์) — ตอนนี้ "Execute as: Me" จะหมายถึงตัวเจ้าของห้องพักเอง
ทำให้ระบบเป็นอิสระ ไม่ผูกกับบัญชีผู้พัฒนาในระยะยาว
4. **ถ้าต้องแก้บั๊ก/อัปเดตโค้ดในอนาคต**: ผู้พัฒนาขอแค่สิทธิ์ "Editor" กลับมาเพิ่ม (ไม่ต้องเป็น Owner อีก)
เพื่อแก้ไขโค้ดต่อได้โดยไม่ต้องโอนความเป็นเจ้าของกลับไปกลับมา

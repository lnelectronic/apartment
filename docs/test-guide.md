# คู่มือทดสอบระบบ App-Room

## ขั้นตอนเตรียมข้อมูลทดสอบ (Developer ทำก่อนส่ง link)

1. เปิด Apps Script Editor: https://script.google.com/d/1uP1yiroqBv4hInGJ1jbQxRyznieX4QWczv4RhLok8qUFKkNZIQ0dHiro/edit
2. รัน `setupSheets()` — สร้าง Sheet ครั้งแรก (ถ้ายังไม่ได้ทำ)
3. รัน `seedTestData()` — ใส่ข้อมูลจำลอง
4. ส่ง Web App URL และ Admin Password ให้เจ้าของหอพัก

ก่อนส่งมอบงานจริง: รัน `clearTestData()` แล้วให้แอดมินกรอกข้อมูลจริงเอง

## Checklist ทดสอบ

### 1. meter.html — หน้าพนักงานจดมิเตอร์ (ไม่เห็นยอดเงิน)

URL: `[Web App URL]?page=meter`

| ทดสอบ | วิธี | ผลที่คาดหวัง |
|-------|------|------------|
| ห้องไม่มีในระบบ | กรอก `C999` | แสดง error "ไม่พบห้อง" |
| ห้องว่าง | กรอก `C202` | แสดง error "ห้องว่าง ไม่มีผู้เช่า" |
| ห้องจดแล้วเดือนนี้ | กรอก `C201` | บล็อก "จดแล้ว ติดต่อแอดมิน" |
| ห้องค้างจ่าย | กรอก `R15` | แสดง สถานะ "ค้างจ่าย" + มิเตอร์เริ่ม ไฟ=560 น้ำ=218 |
| จดปกติ | กรอก `C101` | แสดง สมชาย, ไฟเริ่ม=1050, น้ำเริ่ม=515 กรอก ถึง → บันทึก |
| ห้องไม่มีประวัติ | กรอก `RC1` | ไฟเริ่ม=1500, น้ำเริ่ม=700 (ใช้ค่า init) |

### 4. ทดสอบ ค้างเดือนก่อน (สำคัญมาก)

**ขั้นตอน:**
1. เปิด staff.html → กรอก `C101` → กรอกมิเตอร์ → บันทึก
   - ยอด 6/2026 ต้องมี **ค้างเดือนก่อน = 400**
2. เปิด staff.html → กรอก `R15` → กรอกมิเตอร์ → บันทึก
   - ยอด 6/2026 ต้องมี **ค้างเดือนก่อน = 3,980**
3. เปิด admin.html → C201 → จ่ายจริง = **3,000** (จ่ายไม่ครบ 4,730)
4. เปิด staff.html → กรอก `C201` เดือน 7/2026
   - ค้างเดือนก่อน ต้องเป็น = 4,730 - 3,000 = **1,730**
